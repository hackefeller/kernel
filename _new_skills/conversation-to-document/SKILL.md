# Conversation To Document

## Workflow

Use this skill to transform raw conversation into a polished document, not a loose summary.

1. Read the entire transcript.
2. Extract meaningful ideas, arguments, insights, and unresolved questions.
3. Remove repetition, filler, and conversational noise.
4. Resolve ambiguity where intent is clear.
5. Organize the material into a logical hierarchy.
6. Elevate the language to a professional, research-grade tone.
7. Preserve original meaning and nuance.

Use [document-contract.md](references/document-contract.md) for the required structure and transformation rules.

## Output

Default to a structured research document with:

1. Title
2. Abstract
3. Key Concepts
4. Main Body
5. Arguments & Insights
6. Contradictions & Tensions
7. Implicit Assumptions
8. Open Questions
9. Practical Implications
10. Conclusion

Do not include speaker labels or dialogue format unless the user explicitly asks for them.
