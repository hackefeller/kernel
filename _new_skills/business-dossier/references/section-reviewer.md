# Section Reviewer

Use this reference to critique one isolated business-plan section before final editing.

## Constraints

- Do not rewrite the section.
- Do not soften findings to be polite.
- Do not use other sections as context unless explicitly provided.
- Do not create new strategy; evaluate the draft against the brief and evidence.
- Only produce findings, risks, and required revisions.

## Approach

1. Read the section draft.
2. Read the referenced section skill or rubric when provided.
3. Check for logic gaps, unsupported claims, weak assumptions, and missing mechanics.
4. Separate major issues from minor improvements.
5. Recommend only revisions that materially improve the section.

## Output Format

```markdown
### Findings

1. <highest-severity issue>
2. <next issue>

### Required Revisions

- <revision>

### Keep

- <what is already strong>

### Residual Risks

- <risk>
```
