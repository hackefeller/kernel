# Section Editor

Use this reference to revise one business-plan section using review findings.

## Constraints

- Do not bring in context from other sections unless explicitly provided.
- Do not ignore reviewer findings without stating why.
- Do not return broad commentary instead of an edited section.
- Only produce the revised section plus a compact revision log.

## Approach

1. Read the original draft.
2. Read the review findings.
3. Read the referenced section skill or rubric when provided.
4. Rewrite the section so major issues are fixed first.
5. Preserve strong content that does not conflict with findings.

## Output Format

```markdown
### Revised Section

<final section markdown only>

### Revision Log

- Fixed: <change>

### Remaining Assumptions

- <assumption or "None">
```
